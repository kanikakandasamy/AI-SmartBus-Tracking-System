# System Design

## Overview

The system design defines the architecture and workflow of the AI Smart Bus Tracking and Notification System. It illustrates how different components interact to provide real-time bus tracking, estimated arrival time (ETA), and notification services. The design focuses on simplicity, scalability, security, and efficient communication between users and the system.

---

## Design Components

### Use Case Diagram
Represents the interactions between the users (Student, Driver, and Administrator) and the system functionalities.

### Entity Relationship (ER) Diagram
Shows the database structure, entities, attributes, and relationships used in the system.

### Data Flow Diagram (DFD)
Illustrates how data moves through the system, including user requests, GPS updates, notifications, and database interactions.

### System Architecture
Displays the overall architecture of the application, including the frontend, backend, database, GPS service, and notification service.

---

## Design Objectives

- Provide a scalable architecture.
- Ensure secure communication between users and the system.
- Support real-time GPS tracking.
- Enable efficient database management.
- Deliver a responsive and user-friendly interface.
